SAE - Projet d'application répartie

Map Météo

Mise en situation :

Réalisation d'une application permettant de choisir un restaurant sur une map et d'y faire une réservation

Schéma de l'architecture du projet

Le service central

Il joue plusieurs rôles dans notre application. Il permet de rassembler les différentes données comme les restaurants de Nancy ou les établissements supérieurs au travers de différents services RMI qui s'enregistrent sur lui. Ensuite grâce à son implémentation en serveur HTTP, il rend ces données accessibles à la partie web JS, il fait donc office de proxy.

Le site web

Il propose une carte interactive permettant l’accès et l’utilisation des données du service central. En complément, il envoie une requête sur une autre API afin de connaître les différents emplacements de station de vélo en libre service et des travaux en cours toujours sur Nancy.

Le service de restauration

Ce service implémente la connexion à une base de données Oracle. Il doit s’enregistrer auprès du service central afin que ce dernier puisse l’utiliser pour récupérer ses données. Sa base de données est hébergée sur le serveur de l’IUT, ce qui lui permet d’être accessible depuis tout le bâtiment. L’accès à la liste des restaurants ou encore la possibilité de réserver dans l'un de ces derniers sont des fonctionnalités spécifiques à ce service.

Le service d’établissements supérieurs

Ce service implémente un client HTTP Java. Il doit s’enregistrer auprès du service central afin que ce dernier puisse l’utiliser pour récupérer ses données. Afin de contourner le blocage des données à la lecture depuis les ordinateurs de l’IUT, il est obligatoire d’appliquer le proxy de l’IUT et d’obliger la redirection web lorsqu’elle est possible.
